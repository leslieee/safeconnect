6 Example
Requirement
A remote user wants to access internal resources of the company using an iOS device, and data transmission is required to be secure.
Solution
The remote user can install SSL VPN Client on the iOS device. Through SSL VPN Client, the user can establish an SSL VPN tunnel with the SSL VPN server. In addition, mutual authentication is enabled on the SSL VPN server which requires SSL VPN Client to pass both user name/password and certificate authentication.
Configuration Summary
Configure the following on the SSL VPN server:
Configure Interfaces
Create an IP Address Pool
Create an SSL VPN User
Create an SSL VPN User Group
Create SSL VPN Resources
Create an SSL VPN Tunnel
Create an Access Policy
Configure the following on the iOS device of the remote user:
Import Personal Certificate
Create an SSL VPN Connection
Configuration Steps
Please configure the SSL VPN server according to the following parameter information. For other parameters that are not provided, please use their default values.
Configure Interfaces
1.
Log on to the server WebUI. Choose Network > Interfaces and set the interfaces eth-s1p1 and eth-s1p2 as follows:
eth-s1p1
Active: On
Mode: Layer 3
Static IP: 192.168.1.1/24
eth-s1p2
Active: On
Mode: Layer 3
Static IP: 202.118.1.1/24
2.
Click the Save button.
Create an IP Address Pool
1.
Choose VPN > IP Address Pools.
2.
Create New to create the following IP address pool:
Name: pool
Start IP Address: 30.1.1.1
End IP Address: 30.1.1.100
From the IP address pool, the SSL VPN server will choose an IP address and allocate it to the remote user. The remote user uses this IP address to establish an SSL VPN connection with the server. The addresses in the IP address pool should not overlap with the addresses in the subnet behind the server.
3.
Click the Save button.
Create an SSL VPN User
1.
Choose System > Authentication > Users.
2.
Create New to create an SSL VPN user named Alice. When establishing an SSL VPN connection with the server, the remote user uses this name to log in.
Name: Alice
SSL VPN: Check
Password: 123456
Confirm Password: 123456
IP Address Pool: pool
If the remote user already bought an OTP token and the OTP authentication is required, please choose System > Authentication > OTP Tokens to create an OTP token in advance. Then create the user Alice. Check OTP Authentication and bind Alice to the created OTP token. When establishing an SSL VPN connection with the server, the remote user should enter the OTP password.
3.
Click OK.
4.
Click the Save button.
Create an SSL VPN User Group
1.
Choose VPN > SSL VPN > User Groups.
2.
Create New to create an SSL VPN user group and allocate Alice to the group.
Name: group
Selected Users: Alice
3.
Click OK.
4.
Click Authentication Method and click Mutual Authentication.
5.
Click OK.
6.
Click the Save button.
Create SSL VPN Resources
1.
Choose VPN > SSL VPN > Resources.
2.
Create New to create the following resources:
Name: web-http
Type: HTTP
Address: 192.168.1.10
Port: 80
Name: web-https
Type: HTTPS
Address: 192.168.1.20
Port: 443
Name: subnet
Type: Subnet
IP Address: 192.168.1.0
Mask Length: 24
3.
Click OK.
4.
Click the Save button.
Create an SSL VPN Tunnel
1.
Choose VPN > SSL VPN > SSL VPN Tunnels > Tunnels.
2.
Create New to create an SSL VPN tunnel which is used for data transmission between the remote user and the server.
Name: sslvpn-tunnel
User Group: group
Outgoing Interface: eth-s1p2 (the server interface connecting to the Internet)
Local IP Address: 202.118.1.1 (the IP address of eth-s1p2)
Resource List (internal resources allowed to access):
web-http
web-https
subnet
3.
Click OK.
4.
Click the Save button.
Create an Access Policy
1.
Choose Firewall > Access Policies.
2.
Create New to create an access policy allowing the remote user to access the company network.
Name: permit-alice
Source IP Address: 30.1.1.1-30.1.1.100 (IP addresses set in the IP address pool)
Destination IP Address: 192.168.1.0/24 (the company network allowed to access)
Action: Permit
3.
Click OK.
4.
Click the Save button.
Import Personal Certificate
Connect your iOS device to your local PC. Open iTunes and click the iOS device icon. Choose Applications on the left menu tree. At the bottom of the Applications page, click the Neusoft SSL VPN software in the Share File area. Click Add to import your personal certificate to the iOS device.
Create an SSL VPN Connection
1.
In the login page of SSL VPN Client, enter the following information:
Connection Name: sslvpn-tunnel (SSL VPN tunnel name)
Server IP: 202.118.1.1 (the IP address of the server interface eth-s1p2)
User Name: Alice (SSL VPN user name)
Password: 123456 (password set for Alice)
2.
Click the icon Disconnected. If the OTP authentication is also required, you need to enter the OTP password.
3.
When the icon becomes Connected, SSL VPN Client has successfully established an SSL VPN connection with the server. Then, the remote user can access the company resources behind the server through the SSL VPN tunnel.
4.
For Web resources, you can click the Resources tab in SSL VPN Client and click a specific HTTP or HTTPS resource to access it directly.
