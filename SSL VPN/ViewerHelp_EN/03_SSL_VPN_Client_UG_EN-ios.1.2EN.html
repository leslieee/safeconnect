2 Connection
After installing SSL VPN Client, click to start it and enter the Connection page.
The SSL VPN server requires either one-way authentication or mutual authentication when SSL VPN Client is establishing a VPN connection with the server. One-way authentication only needs the user name/password of SSL VPN Client. Mutual authentication needs not only user name/password of SSL VPN Client but also the certificate authentication. If mutual authentication is required, please save your personal certificate to your iOS device and import it to SSL VPN Client in advance. Then create an SSL VPN connection.

2.1 Import a Personal Certificate
Connect your iOS device to your local PC. Open iTunes and click the iOS device icon. Choose Apps on the left menu tree. At the bottom of the Apps page, click the Neusoft NetEye SSL VPN Client software in the Apps area. Click Add in the Documents area to import your personal certificate to the iOS device.
2.2 Create a Connection
1.
On the login page of SSL VPN Client, enter the following information:
Connection Name—SSL VPN connection name. It is the name of the SSL VPN tunnel which is created on the SSL VPN server.
Server IP—SSL VPN server IP address or domain name. It is the IP address or domain name of the SSL VPN server interface used to establish an SSL VPN connection with a remote user.
User Name—Remote user name. It is the name of the SSL VPN user which is created on the SSL VPN server, used to identify the remote user who wants to establish an SSL VPN connection with the SSL VPN server.
Password—Remote user password. It is set when an SSL VPN user is created on the SSL VPN server.
2.
Click Disconnected. If mutual authentication is required, you need to enter the password of your personal certificate
Disconnected—Indicates SSL VPN Client has not established a connection with an SSL VPN server. Click this icon to connect to the server. If the OTP authentication is also required, you need to enter the OTP password after clicking Disconnected.
Connecting—Indicates SSL VPN Client is connecting to the server. You can click this icon to cancel connecting.
Connected—Indicates SSL VPN client has successfully connected to the server. Communication can be protected. Click this icon to disconnect the connection.
2.3 Delete or Choose a Connection
SSL VPN Client can record the information of established connections, which can be used directly or deleted.
1.
In Disconnected status, click , and click Connection History. You can see the connections added.
2.
Choose the name of a connection. Click Delete to delete the connection or click Select to use it.
